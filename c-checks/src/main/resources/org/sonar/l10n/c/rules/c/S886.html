Why is this an issue?

For loops in C are highly flexible, but this flexibility can lead to complex and error-prone code.When loop control logic is mixed with business logic, the code becomes difficult to understand and maintain.

A well-structured for loop should clearly separate:

  • Initialization
  • Condition
  • Update

Only loop control variables should appear in these expressions.

Common Problems

  • Modifying loop counter inside the loop body
  • Updating variables inside the condition
  • Using external variables (pseudo-counters) to control loop

Noncompliant Code Example

for (int i = 0; i < 10; i++) {
    i++; // Noncompliant
}

for (int i = 0; i++ < 10; i++) { // Noncompliant
}

for (int i = 0, j = 0; i < 10; i += j) { // Noncompliant
    j = i + 1;
}

Compliant Code Example

for (int i = 0; i < 10; i++) {
    doSomething(i);
}
