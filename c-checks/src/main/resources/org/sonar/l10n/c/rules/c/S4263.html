Why is this an issue?

C provides two forms of include directives with different search strategies:

  • #include <header> - Searches system/standard include paths (portable)
  • #include "header" - Searches current directory first, then system paths (non-portable)

Using quoted includes (`#include "header"`) relies on a non-portable search strategy that depends on:

  • The current working directory
  • Compiler-specific search paths
  • Build system configuration

This can cause code to behave differently across different platforms, development environments, or build configurations, making it less portable and maintainable.

Example:

    // Noncompliant 
    #include "stdio.h"
    #include "math.h"
    #include "string.h"
    
    // Compliant - angle-bracket includes (portable search)
    #include <stdio.h>
    #include <math.h>
    #include <string.h>
    
    // Quoted includes are acceptable for project-local headers
    #include "my_local_header.h"  // OK 
    #include "../lib/utils.h"     // OK
