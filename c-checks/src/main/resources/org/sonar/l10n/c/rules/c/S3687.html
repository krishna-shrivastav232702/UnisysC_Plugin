Why is this an issue?

Declaring a local variable with the volatile keyword is almost always unnecessary and prevents the compiler from performing crucial optimizations.

The volatile qualifier tells the compiler that a variable's value may be changed by something outside the control of the code (like a hardware interrupt or a different thread). However, local variables are stored on the stack and are not visible to external hardware or other threads unless their address is explicitly shared.

void compute() {
  volatile int status = 0;  // Noncompliant
