Why is this an issue?

In character and string literals, a backslash introduces an escape sequence. If the sequence that follows is not one of the language-defined escapes, the literal becomes unclear, non-portable, and may not be interpreted as intended by the compiler.

Only defined escape sequences and universal character names should follow \ inside a non raw-string literal. Typical valid forms include:

  • simple escapes such as \n, \t, \\, \", and \'
  • octal escapes such as \123
  • hexadecimal escapes such as \x41
  • universal character names such as \u00E9 and \U00000041

Invalid sequences like \k, \8, or incomplete universal character names can lead to parsing problems and make the code harder to understand and maintain. 

void test() {
    const char *invalid = "\k";       /* Noncompliant */
    const char *invalid2 = "\8";      /* Noncompliant */
    const char *invalid3 = "\u12";    /* Noncompliant */
}

How to fix it

Replace the invalid sequence with a defined escape sequence, a valid universal character name, or the literal character that was actually intended. 

void test() {
    const char *lineBreak = "\n";         /* Compliant */
    const char *letterA = "\x41";         /* Compliant */
    const char *accented = "\u00E9";      /* Compliant */
    const char *plainText = "k";          /* Compliant */
}
