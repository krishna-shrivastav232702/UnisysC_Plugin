Why is this an issue?

Functions that never return (through infinite loops, program termination, or exceptions) should be explicitly declared with the noreturn attribute. This provides important information to the compiler and readers of the code.

Using the noreturn attribute enables:

  • Better compiler optimizations
  • Clearer intent and documentation
  • Detection of unreachable code after function calls
  • Accurate control flow analysis

Example:

    // Noncompliant - function never returns but not marked as noreturn
    void fatalError(const char *msg) {
        fprintf(stderr, "Fatal error: %s\n", msg);
        exit(1);
    }
    
    void infiniteLoop() {
        while (1) {
            printf("Running...\n");
        }
    }
    
    // Compliant - properly marked with noreturn
    _Noreturn void fatalError(const char *msg) {
        fprintf(stderr, "Fatal error: %s\n", msg);
        exit(1);
    }
    
    _Noreturn void infiniteLoop() {
        while (1) {
            printf("Running...\n");
        }
    }
    
    // Compliant - using noreturn macro (if defined)
    #define NORETURN _Noreturn
    
    NORETURN void handleShutdown() {
        cleanup();
        exit(0);
    }
