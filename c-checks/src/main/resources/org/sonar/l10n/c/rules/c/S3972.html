Why is this an issue?

Conditional expressions and their operators should start on new lines to improve code readability and clarity. Placing conditional logic on the same line as other code can make expressions hard to follow and prone to misunderstanding.

Starting conditionals on new lines:

  • Improves visual separation and clarity
  • Makes code easier to scan and understand
  • Reduces line length, improving readability
  • Makes diffs and code reviews clearer
  • Follows consistent coding conventions

Example:

    // Noncompliant - conditional continues on same line
    if (condition1 && condition2 &&
        condition3) {
        doSomething();
    }
    
    // Noncompliant - multiple conditions crammed together
    if (x > 0 && y < 10 && z == 5)
        doSomething();
    
    // Compliant - conditional operators start on new lines
    if (condition1 && condition2 &&
        condition3) {
        doSomething();
    }
    
    // Compliant - clear separation with proper indentation
    if (x > 0
        && y < 10
        && z == 5) {
        doSomething();
    }
    
    // Compliant - logical grouping with line breaks
    if (isValid
        && !isExpired
        && hasPermission) {
        execute();
    }
