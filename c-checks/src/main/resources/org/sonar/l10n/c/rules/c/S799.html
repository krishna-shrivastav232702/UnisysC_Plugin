Why is this an issue?

Long identifiers (variable names, function names, etc.) reduce code readability and can cause portability issues. The C standard specifies that at least the first 31 characters of external identifiers and internal identifiers should be significant for most implementations.

Identifiers longer than 31 characters:

  • May be truncated by some compilers, causing name collisions
  • Reduce code readability on screens with limited width
  • Make line wrapping more likely
  • Can cause portability issues across different platforms
  • Are often unnecessarily verbose

Example:

    // Noncompliant - identifier too long
    int this_is_a_very_long_variable_name_that_exceeds_31_characters = 0;
    
    void this_function_has_an_extremely_long_name_that_is_not_portable() {
        // ...
    }
    
    struct this_struct_name_is_way_too_long_and_should_be_shortened {
        int value;
    };
    
    // Compliant - identifiers within 31 character limit
    int long_variable_name = 0;
    
    void long_function_name() {
        // ...
    }
    
    struct long_struct_name {
        int value;
    };
    
    // Compliant - clear but concise names
    int total_employee_count = 0;
    void calculate_salary() { }
    struct employee_record { };
