Why is this an issue?

Declaring multiple variables in a single statement (e.g., int a, b;) reduces code readability and can lead to confusion, especially when pointers or initializations are involved.

For example, in int* a, b;, a is a pointer to an integer, but b is a regular integer. This subtle distinction is a frequent source of bugs.

In the following code:

int a, b; // Noncompliant

How to fix it

Break the declaration into multiple lines. This makes the code clearer and allows for better commenting of each variable.

int a;
int b;
