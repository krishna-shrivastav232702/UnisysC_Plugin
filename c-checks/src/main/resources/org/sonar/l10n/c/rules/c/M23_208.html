Why is this an issue?

Using certain characters inside a header file name can lead to implementation-defined behavior and reduce portability.

The following characters and sequences must not appear in a header file name:

  • ' (single quote)
  • " (double quote inside the name)
  • \ (backslash)
  • /* (start of comment)
  • // (line comment sequence)

Noncompliant code example

#include "fi'le.h"      // Noncompliant
#include "path\\file"   // Noncompliant
#include "file/*name.h" // Noncompliant

Compliant solution

#include "file.h"       // OK
#include "path/file.h"  // OK
#include <stdio.h>      // OK
