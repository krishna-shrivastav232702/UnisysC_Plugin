Why is this an issue?

C provides both logical and bitwise operators, which have similar symbols but very different behaviors:

  • && (logical AND) vs. & (bitwise AND)
  • || (logical OR) vs. | (bitwise OR)
  • ! (logical NOT) vs. ~ (bitwise NOT)

Using the wrong operator can lead to:

  • Unexpected behavior in conditional logic
  • Incorrect boolean results
  • Logic errors that are difficult to detect
  • Different results when boolean result is expected vs. integer result

Use logical operators (`&&`, `||`, `!`) for boolean conditions and control flow. Use bitwise operators (`&`, `|`, `~`) only when bit manipulation is explicitly intended.

Example:

    int a = 5;   // binary: 0101
    int b = 3;   // binary: 0011
    
    // Noncompliant - using bitwise AND in boolean context
    if (a & b) {  // Bitwise AND: 0101 & 0011 = 0001 (1)
        printf("Condition true\n");  // Result is 1, acts as true
    }
    
    // Noncompliant - confusion between logical and bitwise
    if (a | b) {  // Bitwise OR: 0101 | 0011 = 0111 (7)
        printf("Condition true\n");  // Result is 7, acts as true
    }
    
    // Noncompliant - bitwise operation in boolean expression
    if ((x & 0xFF) && (y & 0xFF)) {  // Confusing mix
        printf("Both have lower byte set\n");
    }
    
    // Compliant - using logical AND for boolean logic
    if (a && b) {  // Logical AND: true if both are non-zero
        printf("Both a and b are non-zero\n");
    }
    
    // Compliant - using logical OR for boolean logic
    if (a || b) {  // Logical OR: true if either is non-zero
        printf("Either a or b is non-zero\n");
    }
    
    // Compliant - clear intent with bitwise operators
    int result = a & b;  // Bitwise AND - explicit bit manipulation
    printf("Bitwise AND result: %d\n", result);
    
    // Compliant - checking specific bits
    if ((flags & MASK) != 0) {  // Check if specific bits are set
        printf("Flag is set\n");
    }
