Why is this an issue?

The "dangling else" problem occurs when an `else` statement's association with an `if` statement is ambiguous. This is particularly problematic when `if` statements are nested or when braces are omitted. The problem arises because C's grammar associates an `else` with the nearest unmatched `if`, which may not be the programmer's intent.

Ambiguous `else` associations lead to:

  • Logic errors and unexpected program behavior
  • Code that is difficult to understand and maintain
  • Bugs that are hard to detect and debug
  • Different interpretations by different programmers

Example:

    // Noncompliant - ambiguous else association
    if (a > 0)
        if (b > 0)
            printf("Both positive\n");
    else
        printf("a is not positive\n");  // Which if does this else belong to?
    
    // Noncompliant - else ambiguity in complex conditions
    if (condition1)
        statement1();
    else if (condition2)
        statement2();
    else if (condition3)
        statement3();  // Unclear structure when braces are omitted
    
    // Compliant - clearly matched with braces
    if (a > 0) {
        if (b > 0) {
            printf("Both positive\n");
        } else {
            printf("b is not positive\n");
        }
    } else {
        printf("a is not positive\n");
    }
    
    // Compliant - explicit braces clarify intent
    if (condition1) {
        statement1();
    } else if (condition2) {
        statement2();
    } else if (condition3) {
        statement3();
    } else {
        statement4();
    }
    
    // Compliant - single statement with braces
    if (x > 0)
        y++;
    else
        y--;
