Field names should comply with a naming convention

A naming convention defines rules for naming variables, functions, and fields to improve readability and consistency.

This rule checks that field names follow a given regular expression.

Noncompliant code example

struct MyStruct {
  int my_field; // Noncompliant
  int Field;    // Noncompliant
};

Compliant solution

struct MyStruct {
  int myField; // Compliant
};

Why is this an issue?

  • Inconsistent naming reduces readability
  • Harder to maintain in large codebases
  • Breaks team coding standards

How to fix it

  • Follow a consistent naming pattern
  • Use camelCase or the project’s defined convention
  • Ensure names match the configured regex
